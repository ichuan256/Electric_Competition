# 双源幅值、直流偏置与相位同步总方案

## 1. 文档目的

本文档用于联调 AD9910 DDS、FPGA DAC1、FPGA DAC2、Blue 与 Black，解决以下三类问题：

1. DDS、DAC1、DAC2 设置相同幅值时，加法器后实际幅值差别很大。
2. DDS 输出含有约 0.5 V 直流偏置，需要用 FPGA 控制的 DAC 偏置抵消。
3. DDS—DAC1、DDS—DAC2 以及 DAC1—DAC2 之间的初始相位不固定，普通更新时还可能出现额外相位跳变。

本方案优先保证三个波形源的相对幅值和相对相位正确。加法器后的绝对输出电压由后级手动可调放大器设置，不在 FPGA 中硬绑定为某个最终电压。

## 2. 已确认的硬件结构

```text
AD9910 DDS ──缓冲/分配──┬──加法器1 ──可调放大── CH1 电压输出
                       │       ↑
                       │      DAC1
                       │
                       └──加法器2 ──可调放大── CH2 电流输出
                               ↑                      │
                              DAC2                   固定 5 Ω
                                                      │
                                                     GND
```

- CH1 最终是电压输出。
- CH2 通过固定 5 Ω 电阻把电压换算为电流：`I = V / 5 Ω`。
- CH2 先不修改模拟网络，由软件限制量程。
- DDS 为两路共享波形源，因此 DDS 成分在 CH1、CH2 上不能同时拥有两个独立频率。

## 4. 幅值归一化方案

### 4.1 统一的上层语义

建议把用户设置的幅值解释为“加法器后的目标相对 Vpp”，记为 `A_target_vpp`。后级放大倍数可调，所以这个值只用于保证各波形源的相对幅值。

换算式：

```text
DDS_内部设置  = A_target_vpp / 0.21321
DAC1_内部设置 = A_target_vpp / 2.51895
DAC2_内部设置 = A_target_vpp / 2.34966
```

当 `A_target_vpp = 0` 时，必须直接输出零幅值，不应把线性回归的小截距换算成非零波形。

### 4.2 推荐联调基准幅值

先以加法器后 `500 mVpp` 为单波形基准：

```text
DDS_框值  ≈ 2.345
DAC1_框值 ≈ 0.199
DAC2_框值 ≈ 0.213
```

两个同频正弦波在此幅值下应满足：

| 相位差 | 理论合成 Vpp |
|---:|---:|
| 0° | 约 1.000 Vpp |
| 90° | 约 0.707 Vpp |
| 180° | 接近 0 Vpp |

### 4.3 责任划分

- Black/Blue：把统一目标幅值换算为 DDS ASF 和 FPGA `amplitude_code`。
- 用户界面量程映射：内部 0～600 mVpp 对应 CH1 的 0～20000 mVpp；对应 CH2 的 0～250 mApp。CH2 按固定 5 Ω 负载换算，0.25 App 对应负载两端 1.25 Vpp；两路满量程均由后级手动校准。
- FPGA：按收到的各分量幅值码乘法、宽位累加和饱和，不再自行套用“8191 一定等于某个最终电压”的假设。
- DAC1 和 DAC2 分别保留幅值标定系数，不得共用同一个物理电压换算常量。

### 4.4 FPGA 饱和要求

```verilog
wide_sum = component_0 + component_1 + component_2 + component_3
         + applied_offset;
dac_data = saturate_to_signed_14bit(wide_sum);
```

- 中间累加使用至少 18～20 位有符号数。
- 输出饱和到 `-8192～8191`，禁止直接截断低 14 位造成回绕。
- 发生饱和时置 `clipping` 标志并增加计数器。
- DDS 是在 DAC 之后进行模拟相加，FPGA 不能仅靠数字饱和保护最终运放；Blue/Black 必须同时校验 DDS 与 DAC 的最坏情况峰值之和。

## 5. DDS 直流偏置补偿

### 5.1 实测结论

- DDS 通路自身平均值约为 487.6 mV，随 DDS 幅值变化很小。
- FPGA 已能接收并应用 BIAS，但加法器为反相和缩放通路，当前框值与实际平均值的方向和比例都不一致。
- 两路均已完成 `BIAS=-500、-100、0、+100、+500` 五点标定。

| BIAS 框值/当前发送码 | -500 | -100 | 0 | +100 | +500 |
|---:|---:|---:|---:|---:|---:|
| DAC1 实际平均值/mV | 667 | 529 | 494 | 459 | 319 |
| DAC2 实际平均值/mV | 660 | 533 | 495 | 463 | 320 |

DAC1 线性拟合：

```text
Vavg_ch1_mV = 493.6 - 0.3480769 × offset_code
R² = 0.99998
user_code_per_mV = -2.87293
dds_cancel_code_ch1 = +1418
```

DAC2 线性拟合：

```text
Vavg_ch2_mV = 494.2 - 0.3403846 × offset_code
R² = 0.99889
user_code_per_mV = -2.93785
dds_cancel_code_ch2 = +1452
```

两路偏置斜率相差约 2.2%，因此必须使用两套独立换算系数和补偿码。

### 5.2 偏置组成

```text
DAC1：
user_offset_code = round(-2.87293 × user_bias_mV)

DAC2：
user_offset_code = round(-2.93785 × user_bias_mV)

DDS 开启：
applied_offset_code[DAC1] = 1418 + user_offset_code[DAC1]
applied_offset_code[DAC2] = 1452 + user_offset_code[DAC2]

DDS 关闭：
applied_offset_code[ch] = user_offset_code[ch]
```

其中：

- `user_bias_mV` 是用户要求的加法器后、可调放大器前的最终直流偏置。
- `user_offset_code` 是根据通道独立斜率反号换算后的 FPGA 原始码。
- `dds_cancel_code` 是隐藏的硬件标定值，不在普通 OFFSET 输入框中显示。
- DDS 使能时加入 `dds_cancel_code`。
- DDS 关闭时 `dds_cancel_code = 0`，否则会出现反向直流偏置。

必须保证：

```text
sum(abs(component_code)) + abs(applied_offset_code) <= 8191
```

### 5.3 实施后验证

1. DDS 开启、用户 BIAS=0 时，DAC1 自动发送约 `+1418`，DAC2 自动发送约 `+1452`。
2. 两个加法器输出平均值应接近 0 V。
3. BIAS 设置为 `-500、-100、0、+100、+500 mV`，实际平均值应同方向变化，不再反号。
4. DDS 关闭时必须撤销 `+1418/+1452` 补偿码。
5. 改变 DDS ASF 重复检查；若零点随 ASF 明显变化，再升级为 `dds_cancel_code = k0 + k1×ASF`。

## 6. 相位问题的分类

相位问题必须分为三层，不能只靠修改相位字同时解决。

### 6.1 频率不完全相同造成持续漂移

AD9910 与 FPGA 已使用同源时钟。频率字必须继续使用同一次取整：

```text
ad9910_ftw = round(f × 2^32 / 1 GHz)
fpga_ftw  = ad9910_ftw × 10
```

禁止 AD9910 和 FPGA 分别根据理想频率独立取整。这一层解决“持续相对漂移”，但不保证上电后的初始相位固定。

### 6.2 普通更新误清相位累加器造成跳变

修改前 Black 工程 `Black/User/Src/SpectrumSystem_User.c` 发送：

```c
payload[pos++] = 0x09U;
```

`0x09` 使 `commit_flags.bit0=1`，会请求 FPGA 在 COMMIT 时清零相位累加器。AD9910 的 I/O_UPDATE 和 FPGA COMMIT 没有在同一硬件时钟边沿发生，因此普通改 1° 也可能表现为很大的随机相位变化。

本轮单片机工程已修改为：

```c
payload[pos++] = 0x08U;
```

并在 FPGA 中严格实现：

```verilog
if (commit_flags[0]) begin
    phase_acc <= 32'd0;
end
// bit0=0 时保留 phase_acc，不得因普通 COMMIT 清零。
```

连续相位累加器和用户初相必须分离：

```verilog
phase_acc    <= phase_acc + ftw;
output_phase = phase_acc + phase_offset;
```

普通修改幅值、偏置、波形、占空比或 `phase_offset` 时，不清零 `phase_acc`。

### 6.3 上电或同步启动时的初始相位不固定

仅靠串口通信无法让 AD9910 I/O_UPDATE 与 FPGA 相位清零落在同一个确定时钟边沿。如果要求每次上电、重启或“同步开始”后 DDS—DAC1、DDS—DAC2 的初始相位可重复，必须增加硬件同步线。

## 7. FPGA 相位引擎必须实现的内容

### 7.1 DAC1 与 DAC2 同步

1. DAC1、DAC2 使用同一个 100 MHz 采样时钟和同一个 `sample_ce`。
2. 每个通道、每个分量拥有独立的 `phase_acc[ch][slot]`。
3. DAC1、DAC2 的 shadow 配置用同一 `transaction_id` 暂存。
4. `channel_mask=0x03` 且 `commit_flags.bit1=1` 时，两路 active 配置必须在同一个采样边沿原子切换。
5. `commit_flags.bit0=1` 时，所有目标通道的相位累加器在该同一边沿清零。
6. 禁止先更新 DAC1、过一个或多个时钟后再更新 DAC2。

推荐时序：

```verilog
if (sample_ce) begin
    if (atomic_commit) begin
        active_ch1 <= shadow_ch1;
        active_ch2 <= shadow_ch2;
    end

    if (sync_phase_reset) begin
        phase_acc_ch1 <= 32'd0;
        phase_acc_ch2 <= 32'd0;
    end else begin
        phase_acc_ch1 <= phase_acc_ch1 + ftw_ch1;
        phase_acc_ch2 <= phase_acc_ch2 + ftw_ch2;
    end
end
```

实际 RTL 必须处理每通道的多个 slot，上述代码仅表示时序原则。

### 7.2 DDS 与 FPGA 的硬件同步启动

推荐方式：FPGA 作为同步主机，增加一根 `DDS_SYNC_START` 线，直接或经硬件逻辑连到 AD9910 `I/O_UPDATE`。该线不应经过 MCU 软件中断再转发，否则中断延迟和抖动仍会使初始相位不确定。

同步启动顺序：

```text
1. Black 通过 SPI 写入 AD9910 的 FTW、POW、ASF 影子配置，但不立即产生普通 I/O_UPDATE。
2. Black 回报 DDS_ARMED。
3. Blue 向 FPGA 暂存 DAC1、DAC2 配置。
4. FPGA 确认两路 shadow 都 ready。
5. FPGA 在确定的共同时钟边沿：
   - 原子切换 DAC1/DAC2 active 配置；
   - 清零 DAC1/DAC2 相位累加器；
   - 产生 DDS_SYNC_START / AD9910 I/O_UPDATE 脉冲。
6. AD9910 使用数据手册规定的“相位累加器清零/自动清零”配置，使新波形从 POW 对应初相启动。
```

同步脉冲宽度、与 AD9910 `SYNC_CLK` 的建立/保持时间必须按 AD9910 数据手册设计，不得仅按 MCU GPIO 一次翻转的经验值实现。

### 7.3 没有新增硬件同步线时的上限

如果本轮无法增加 `DDS_SYNC_START` 硬件线，可以完成：

- 消除 DDS 与 FPGA 的持续相对漂移。
- 保证 DAC1 与 DAC2 同步启动和相对相位固定。
- 保证普通参数修改不重置 FPGA 相位。
- 在一次上电运行期间，手动校正 DDS 相位后保持稳定。

但无法保证每次上电后 DDS 与 FPGA 的绝对初始相位重复。这是硬件同步问题，不能通过串口延时或固定的软件等待时间可靠解决。

## 8. 固定模拟相位延迟的标定

完成数字同步后，电阻、运放、低通、线缆和 DDS 模块仍会产生固定的模拟相位差。该误差不随机，可以标定。

建议保存：

```text
phase_cal_dds_to_dac1[f]
phase_cal_dds_to_dac2[f]
phase_cal_dac1_to_dac2[f]
```

若 DDS 只有一路并同时送往两个加法器，前两个标定值已可推出 DAC1—DAC2 固定延迟，但仍建议独立测量第三项用于交叉验证。

最终使用：

```text
phase_word_applied
    = phase_word_requested
    + phase_calibration_word[f]
```

所有加法按 32 位自然回绕。标定值可先在 1 kHz、10 kHz、100 kHz、1 MHz 测量；如果带宽内差异明显，使用分段线性查表。

## 9. 协议使用和建议扩展

### 9.1 现有协议必须正确使用

| 标志 | 含义 | 用途 |
|---|---|---|
| `commit_flags.bit0` | 清零 FPGA 相位累加器 | 仅显式同步启动 |
| `commit_flags.bit1` | DAC1/DAC2 原子切换 | 双通道相位同步 |
| `commit_flags.bit2` | 等待 CACHE 构建 | CACHE 模式 |
| `commit_flags.bit3` | 失败保持旧输出 | 所有正式提交 |

普通提交使用 `0x08`。双 DAC 原子普通提交使用 `0x0A`。仅 FPGA 双 DAC 同步清相位可使用 `0x0B`。

### 9.2 增加硬件 DDS 同步线后

建议使用当前未定义的 `commit_flags.bit4` 表示：

```text
bit4 = 本次原子切换边沿同时产生 DDS_SYNC_START 脉冲
```

要求：

- bit4 只有在 DDS 已回报 `DDS_ARMED` 时允许执行。
- bit4 一般与 bit0、bit1、bit3 同时使用。
- 推荐同步启动标志值为 `0x1B`。
- FPGA 必须在返回 ACK 之前确认原子切换和同步脉冲都已完成。

该扩展需要同步更新 `UNIFIED_DUAL_SOURCE_PROTOCOL_V2.md`、Blue、Black 和 FPGA，不得只单独修改 FPGA。

## 10. 建议实施顺序

### 阶段 A：不增加新硬件线也能完成

1. FPGA 确认并修复 `commit_flags.bit0` 条件清零逻辑。
2. Black 普通提交从 `0x09` 改为 `0x08`。
3. FPGA 实现 DAC1/DAC2 同一边沿原子 COMMIT。
4. 使用实测系数完成 DDS、DAC1、DAC2 相对幅值归一化。
5. 完成 CH1/CH2 两个 DDS 直流偏置补偿码标定。
6. 在一次上电周期内测试 DDS—DAC1、DDS—DAC2、DAC1—DAC2 的相对相位稳定性。

### 阶段 B：需要每次上电初相重复时

1. 增加 FPGA→AD9910 `DDS_SYNC_START / I/O_UPDATE` 硬件线。
2. Black 实现 DDS 配置暂存和 `DDS_ARMED` 握手。
3. FPGA 实现 `commit_flags.bit4` 同步脉冲。
4. Blue 组织 DDS ARM、双 DAC STAGE 和一次原子 SYNC COMMIT。
5. 测量并保存固定模拟相位延迟补偿表。

## 11. FPGA 验收项目

### 11.1 DAC1—DAC2

1. 同频、同幅、同相设置后，重复 100 次双通道同步启动，相位差不得随机变化。
2. 只修改幅值、偏置或占空比，两路相位连续。
3. 重发相同普通 COMMIT，相位累加器不清零。
4. 仅在 bit0=1 时两路同步清零。

### 11.2 DDS—DAC1 和 DDS—DAC2

1. 使用同源时钟和同一次 FTW 取整，连续运行时相对相位不漂移。
2. 相位从 0°、1°、2°、3° 递增，每次实测只改变约 1°，不出现大幅随机跳变。
3. 只修改幅值或偏置，相对相位不变。
4. 若安装硬件同步线，重复上电或同步启动 100 次，初始相位误差应保持在项目允许范围内。

### 11.3 幅值与偏置

1. 使用基准 500 mVpp，DDS、DAC1、DAC2 分别单独输出时幅值误差建议先控制在 ±3% 内。
2. 两个 500 mVpp 同频正弦波在 0°、90°、180° 时分别符合 1.000 Vpp、0.707 Vpp、接近 0 Vpp。
3. DDS 开启后 CH1、CH2 加法器输出平均值建议先调到 ±10 mV 内，后续再根据题目精度紧缩。
4. DDS 关闭后必须撤销补偿码，输出不得出现与原 DDS 偏置反向的直流电压。
