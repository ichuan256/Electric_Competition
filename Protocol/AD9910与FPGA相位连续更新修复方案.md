# AD9910 与 FPGA 相位连续更新修复方案

## 1. 文档状态

- 状态：待实施
- 记录日期：2026-07-16
- 适用范围：Black、Blue 与 FPGA 使用统一双路信号源 V2 协议的工程
- 目标：消除修改 AD9910 相位或其他普通参数时，AD9910 与 FPGA 同频波形之间出现的大幅相位跳变

## 2. 现象

AD9910 与 FPGA 已使用同源时钟和匹配的 FTW，同频输出之间不再持续漂移。但是从界面修改 AD9910 相位时，即使只修改 1°，示波器上也可能出现远大于 1° 的相对相位跳变。

## 3. 原因

问题不是 1° 相位字的换算精度，而是两侧相位累加器的更新方式不一致：

1. Black 在用户确认输入后立即更新 AD9910 的 Profile POW，并触发 I/O_UPDATE；AD9910 的相位累加器继续运行。
2. Black 随后向 Blue 发送完整的信号源事务。
3. 当前 `SOURCE_COMMIT` 的 `commit_flags` 固定为 `0x09`。
4. `0x09` 的 bit0 为 1，要求 FPGA 在 COMMIT 切换边沿清零目标通道的相位累加器。
5. AD9910 的 I/O_UPDATE 与 FPGA COMMIT 不在同一个参考时钟边沿发生，因此 FPGA 单独清零会引入与通信时刻有关的大幅静态相位跳变。

相位换算误差不足以造成该现象：

- AD9910：1° 对应 POW 182，实际约为 0.99976°。
- FPGA：1° 对应 32 位相位字，实际约为 1.000000°。
- 两侧换算差约为 0.00024°。

## 4. 需要实施的两处修改

### 4.1 Black：普通提交不再请求清零 FPGA 相位累加器

文件：`Black/User/Src/SpectrumSystem_User.c`

当前代码：

```c
payload[pos++] = 0x09U;
```

修改为：

```c
payload[pos++] = 0x08U;
```

标志含义：

| 值 | bit0 清相位 | bit3 失败保持旧输出 | 用途 |
| --- | --- | --- | --- |
| `0x08` | 否 | 是 | 普通参数更新 |
| `0x09` | 是 | 是 | 明确发起的同步重新起相 |

普通的频率、相位、幅值、偏置、占空比、波形和使能修改均应使用 `0x08`。`0x09` 只保留给以后专门设计的“同步启动”或“相位复位”操作。

### 4.2 FPGA：严格按 `commit_flags.bit0` 控制相位清零

FPGA 处理 `FPGA_COMMIT(0x21)` 时，应满足：

```verilog
if (commit_flags[0]) begin
    phase_acc <= 32'd0;
end
```

当 `commit_flags[0] == 0` 时必须保留当前相位累加器，不能在每次 COMMIT 时无条件执行以下操作：

```verilog
phase_acc <= 32'd0;
```

也不能把普通的相位参数更新直接实现为：

```verilog
phase_acc <= phase_word;
```

推荐将连续运行的相位累加器和用户设置的相位偏移分开：

```verilog
phase_acc    <= phase_acc + ftw;
output_phase = phase_acc + phase_offset;
```

普通 COMMIT 只更新 `ftw` 和 `phase_offset` 等配置；只有 bit0 为 1 时才同步清零 `phase_acc`。

## 5. 不需要修改的部分

- Blue：当前已经把 Black 的 `commit_flags` 原样传给 FPGA，不需要修改。
- AD9910 相位换算：当前 16 位 POW 换算精度满足要求，不需要修改。
- AD9910 普通 Profile 更新：继续保持相位累加器连续运行。

## 6. 实施顺序

1. 先确认或修改 FPGA 的 bit0 条件清零逻辑。
2. 将 Black 的普通 `commit_flags` 从 `0x09` 改为 `0x08`。
3. 编译并烧录 Black；同步更新并烧录 FPGA。
4. Blue 不重新修改，仅确保使用当前 V2 协议固件。

## 7. 验收方法

1. AD9910 和 FPGA 设置为相同频率，确认相对相位不随时间漂移。
2. 相位依次设置为 0°、1°、2°、3°，每次示波器上应只出现约 1° 的相位变化，不得出现大幅随机跳变。
3. 测试 359° 到 0° 的环绕变化，结果应等效为约 1°。
4. 只修改幅值、偏置或占空比时，相对相位不应被重新随机化。
5. 重发相同的 `0x08` COMMIT 时，FPGA 相位累加器不得清零。
6. 单独发送一次明确的 `0x09` 同步命令时，FPGA 才允许按协议清零目标通道相位累加器。

## 8. 限制

该修复解决普通参数修改引起的相位跳变，并保持相位连续。若要求上电后 AD9910 与 FPGA 的绝对相位也固定，则仍需使用共同硬件同步信号，使 AD9910 I/O_UPDATE（或相位累加器清零）与 FPGA COMMIT 落在确定的参考时钟边沿；仅靠两块控制器之间的串口通信无法保证绝对同步。
