# DAC1多波形叠加 UART 通信协议 README

串口参数：115200 baud，8数据位，无校验，1停止位。

## 请求帧

```
A5 5A TARGET COUNT [ENTRY0 ... ENTRYn] OFFSET_L OFFSET_H XOR 5A A5
```

- `TARGET=00`：DAC1，当前版本支持。
- `TARGET=01`：DAC2，已经预留，当前版本不接收。
- `COUNT=00..04`：本帧连续包含的波形槽位数量。
- `XOR`：从 `TARGET` 开始，到 `OFFSET_H` 为止的所有字节逐字节异或。
- 多字节整数全部低字节先发送。

每个波形条目固定13字节：

```
TYPE FTW0 FTW1 FTW2 FTW3 PHASE0 PHASE1 PHASE2 PHASE3 AMP0 AMP1 DUTY0 DUTY1
```

波形类型：

| TYPE | 波形 |
|---:|---|
| 00 | 关闭槽位 |
| 01 | 方波 |
| 02 | 三角波，4096相位位置 |
| 03 | 锯齿波，4096相位位置 |

参数：

- `FTW`：32位频率字，100 MHz时 `FTW=round(Fout*2^32/100000000)`。
- `PHASE`：32位相位偏置。
- `AMP`：14位无符号幅度系数，范围0..8191。
- `DUTY`：16位占空比，方波使用；`8000`表示50%。
- `OFFSET`：14位二进制补码全局偏置。

FPGA不执行饱和限幅。MCU必须保证四个缩放波形与偏置之和始终位于
`-8192..8191`，否则14位截断会回绕。

当前诊断版本上电默认配置为：DAC1槽位0输出1 MHz、满幅、零偏置三角波，
其余三个槽位关闭。该默认波形不依赖UART，用于检查混合器和DAC输出链路。

## 应答帧

只有完整请求帧通过目标、数量、长度、XOR和帧尾检查并提交后，FPGA才返回：

```
5A C0 00 9A
```

四个应答字节以115200 baud连续发送，只包含UART标准停止位，不插入额外
毫秒级间隔。

LED1在收到任意UART字节后约2 Hz持续闪烁，用于确认物理接收链路。PS_LED0
只在完整请求帧通过开头、数据长度、XOR和结尾检查并提交后约1 Hz持续闪烁。
两颗板载LED均为低电平点亮。

每个正确请求帧最多回传一次四字节应答。只有检测到下一帧新的 `A5`起始
字节后，成功应答才会重新使能。

配置接收期间活动输出不变化；只有完整帧提交后，四个槽位和偏置才会在
DAC采样时钟域同时更新。

## 单路验证示例

DAC1槽位0输出1 MHz满幅锯齿波、零相位、零偏置：

```
A5 5A 00 01 03 29 5C 8F 02 00 00 00 00 FF 1F 00 80 00 00 9A 5A A5
```

接收成功应返回：

```
5A C0 00 9A
```
