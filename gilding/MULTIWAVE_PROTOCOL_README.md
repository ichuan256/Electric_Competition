# DAC1多波形叠加 UART 通信协议 README

串口参数：115200 baud，8数据位，无校验，1停止位。

## 请求帧

```
A5 5A TARGET CONTROL [ENTRY0 ... ENTRYn] OFFSET_L OFFSET_H [POINTS_L POINTS_H] XOR 5A A5
```

- `TARGET=00`：DAC1，支持实时DDS 0..4槽和周期缓存0..4槽。
- `TARGET=01`：DAC2，支持实时DDS 0..1槽；不支持缓存模式和多槽叠加。
- `CONTROL[7]`：`0`为实时DDS叠加，`1`为单周期缓存循环输出。
- `CONTROL[2:0]`：本帧连续包含的波形槽位数量，范围0..4。
- `CONTROL[6:3]`：保留，必须为0。
- `POINTS`：仅缓存模式存在，位于偏置之后；16位小端序，有效范围16..4096。
- `XOR`：从 `TARGET` 开始，到负载最后一个字节为止逐字节异或；实时模式
  最后一个字节是 `OFFSET_H`，缓存模式最后一个字节是 `POINTS_H`。
- 多字节整数全部低字节先发送。

每个波形条目固定13字节：

```
TYPE FTW0 FTW1 FTW2 FTW3 PHASE0 PHASE1 PHASE2 PHASE3 AMP0 AMP1 DUTY0 DUTY1
```

波形类型：

| TYPE | 波形 |
|---:|---|
| 00 | 关闭槽位 |
| 01 | 方波 |
| 02 | 三角波，4096相位位置 |
| 03 | 锯齿波，4096相位位置 |

参数：

- `FTW`：32位频率字，100 MHz时 `FTW=round(Fout*2^32/100000000)`。
- `PHASE`：32位相位偏置。
- `AMP`：14位无符号幅度系数，范围0..8191。
- `DUTY`：16位占空比，方波使用；`8000`表示50%。
- `OFFSET`：14位二进制补码全局偏置。

每次完整配置帧成功提交后，四个槽位会在同一个DAC采样时钟边沿同步清零
相位累加器，再从各自的 `PHASE` 偏移开始运行。同频且 `PHASE=0` 时采用
以下统一、可重复的零相位定义：

- 方波：第一个上升沿，随后进入高电平；
- 锯齿波：最低点，随后沿正斜率上升；
- 三角波：零码值，随后沿正斜率上升。

四路波形通过相同的原始码寄存、乘法、缩放和加法流水级逐点相加，槽位编号
不会引入额外流水延迟；增加的流水级只形成所有波形共有的固定输出延时。

### 两种输出模式

实时模式的 `CONTROL` 为 `00..04`，与旧版 `COUNT` 完全兼容。FPGA持续运行
32位DDS，适合任意FTW和没有公共周期的频率组合。每次完整配置提交时，四路
相位累加器同步清零。

缓存模式的 `CONTROL` 为 `80..84`。FPGA收到配置后继续输出旧波形，同时用
备用BRAM计算 `POINTS` 个合成采样点；每个槽位使用三拍完成原始码寄存、DSP
乘法寄存以及缩放/累加，每个点依次完成四槽补码相加和直流偏置。计算完成后
切换BRAM，并永久循环输出完全相同的码序列。最大4096点后台生成时间约492微秒。

建议由MCU选择槽位0作为公共基频并计算：

```
POINTS = round(100000000 / 公共基频)
```

其他槽位最好为公共基频的整数倍。若 `FTW*POINTS` 不是整周期，缓存仍会严格
重复，但缓存首尾可能产生跳变，因此实际频谱会包含量化杂散。超出16..4096
的POINTS会被FPGA拒绝，配置不会提交，也不会返回成功应答。

FPGA不执行饱和限幅。MCU必须保证四个缩放波形与偏置之和始终位于
`-8192..8191`，否则14位截断会回绕。

当前诊断版本上电默认配置为：DAC1槽位0输出1 MHz、满幅、零偏置三角波，
其余三个槽位关闭。该默认波形不依赖UART，用于检查混合器和DAC输出链路。

DAC2收到`TARGET=01`的实时单槽帧后，使用槽位0的FTW、PHASE、AMP、DUTY、
TYPE和全局OFFSET原子更新独立DDS。`COUNT=0`关闭DAC2数字输出。DAC2成功
接收UART配置后，板载KEY1仍可在当前频率、幅度和偏置下切换方波、三角波和
锯齿波。DAC2缓存帧或`COUNT>1`的帧会被拒绝，不提交配置也不返回成功应答。

## 应答帧

只有完整请求帧通过目标、数量、长度、XOR和帧尾检查并提交后，FPGA才返回：

```
5A C0 00 9A
```

四个应答字节以115200 baud连续发送，只包含UART标准停止位，不插入额外
毫秒级间隔。

LED1在收到任意UART字节后约2 Hz持续闪烁，用于确认物理接收链路。PS_LED0
只在完整请求帧通过开头、数据长度、XOR和结尾检查并提交后约1 Hz持续闪烁。
两颗板载LED均为低电平点亮。

每个正确请求帧最多回传一次四字节应答。只有检测到下一帧新的 `A5`起始
字节后，成功应答才会重新使能。

配置接收期间活动输出不变化；只有完整帧提交后，四个槽位和偏置才会在
DAC采样时钟域同时更新。

## 单路验证示例

DAC1槽位0输出1 MHz满幅锯齿波、零相位、零偏置：

```
A5 5A 00 01 03 29 5C 8F 02 00 00 00 00 FF 1F 00 80 00 00 9A 5A A5
```

接收成功应返回：

```
5A C0 00 9A
```

同一个1 MHz锯齿波使用100点周期缓存时，`CONTROL=81`，`POINTS=0064`：

```
A5 5A 00 81 03 29 5C 8F 02 00 00 00 00 FF 1F 00 80 00 00 64 00 7E 5A A5
```

DAC2实时输出1 MHz、半幅锯齿波、零相位、零偏置：

```
A5 5A 01 01 03 29 5C 8F 02 00 00 00 00 00 10 00 80 00 00 6B 5A A5
```
