# Gilding AD9744 信号源设计说明

## 一、FPGA 已实现功能

当前 PL 逻辑包含：

- 32 位相位累加器 DDS；
- 14 位二进制补码输出；
- 正弦波、矩形波、三角波和锯齿波；
- 频率、相位、幅度、直流偏置、矩形波占空比和输出使能控制；
- FPGA 与单片机之间的 UART 配置接口。

上电默认输出仍为 1 MHz 满幅正弦波，方便沿用已经验证过的硬件连接进行调试。

## 二、时钟 IP 核

板载 50 MHz 时钟不直接作为 DAC 输出时钟。`create_clock_ip.tcl` 会创建赛灵思
Clocking Wizard IP `clk_wiz_0`，将 50 MHz 输入时钟转换为 100 MHz 采样时钟。

工程已经包含 `clk_wiz_0.xci`，正常打开工程后可直接生成 IP 输出产品并综合。
如果需要重新创建或覆盖 IP，也可以在 Vivado Tcl Console 中执行：

```tcl
source create_clock_ip.tcl
```

100 MSPS 下，32 位 DDS 的频率分辨率为：

```text
100000000 / 2^32 = 0.023283 Hz
```

单片机按下式计算频率控制字和相位控制字：

```text
FTW = round(输出频率_Hz × 2^32 / 100000000)
PHASE = round((相位角_deg mod 360) × 2^32 / 360)
```

该分辨率可以满足 10 Hz 频率步进和 5° 相位步进。100 MSPS 可覆盖题目要求的
5 MHz 正弦波和 1 MHz 其他波形。20 MHz 正弦波属于发挥部分，仍需实测频谱纯度、
重建滤波器衰减以及后级放大器平坦度。

## 三、UART 控制协议

UART 参数为 115200 bit/s、8 位数据、无校验、1 位停止位，电平为 3.3 V。

单片机发送 7 字节数据帧：

```text
A5 CMD DATA0 DATA1 DATA2 DATA3 XOR
```

`DATA0` 是最低有效字节。`XOR` 是前 6 字节的逐字节异或校验。命令正确执行后，
FPGA 返回：

```text
5A CMD 00 XOR
```

| 命令 CMD | 控制内容 | 32 位数据含义 |
| --- | --- | --- |
| 01 | 频率控制字 | 无符号 32 位 FTW |
| 02 | 相位偏移 | 无符号 32 位相位控制字 |
| 03 | 幅度 | 0～8191，8191 为满幅 |
| 04 | 直流偏置 | 有符号 14 位 DAC 码 |
| 05 | 矩形波占空比 | 0～65535 对应 0～100% |
| 06 | 波形选择 | 0 正弦，1 矩形，2 三角，3 锯齿 |
| 07 | 输出使能 | bit0，1 为输出，0 为零电平 |

题目要求的 0.1%～99.9% 占空比，对应单片机发送值 66～65469。为避免削顶，
单片机界面还应限制：

```text
abs(直流偏置码) + 幅度码 <= 8191
```

FPGA 内部另有最终饱和限制，作为第二重保护。

## 四、DAC 编码和时序

模块原理图中 R24（10 kΩ）已安装并连接 3.3 V，R25 未安装，因此 AD9744 的
MODE 引脚为高电平，数字接口采用二进制补码：

| 模拟电平 | `dac_data[13:0]` |
| --- | --- |
| 最大负值 | `14'h2000`，即 -8192 |
| 零电平 | `14'h0000` |
| 最大正值 | `14'h1FFF`，即 +8191 |

AD9744 要求输入数据建立时间不小于 2.0 ns、保持时间不小于 1.5 ns。FPGA 在
100 MHz 内部采样时钟上升沿更新数据，ODDR 转发的 DAC 时钟在半个周期后产生
上升沿，因此标称建立和保持裕量均约为 5 ns。XDC 已加入相应输出时序约束，最终
仍需在 Vivado Implementation 的时序报告中确认。

## 五、引脚对照表

以下映射严格按照确认后的对照表排列，XDC 中的实际约束与此一致：

| FPGA 管脚 | Verilog 端口 | AD9744 信号 |
| --- | --- | --- |
| R16 | `dac_data[1]` | D2 |
| R17 | `dac_data[0]` | D1 |
| W18 | `dac_data[3]` | D4 |
| W19 | `dac_data[2]` | D3 |
| P15 | `dac_data[5]` | D6 |
| P16 | `dac_data[4]` | D5 |
| Y18 | `dac_data[7]` | D8 |
| Y19 | `dac_data[6]` | D7 |
| T17 | `dac_data[9]` | D10 |
| R18 | `dac_data[8]` | D9 |
| V17 | `dac_data[11]` | D12 |
| V18 | `dac_data[10]` | D11 |
| T16 | `dac_data[13]` | D14 |
| U17 | `dac_data[12]` | D13 |
| Y17 | `dac_sleep` | PD/SLEEP |
| Y16 | `dac_clk` | CLK |
| H15 | `led0` | 板载 LED0 |

单片机 UART 连接：

| FPGA 管脚 | Verilog 端口 | 连接方向 |
| --- | --- | --- |
| K14 | `mcu_uart_rxd` | 单片机 TX → FPGA RX |
| M15 | `mcu_uart_txd` | 单片机 RX ← FPGA TX |

AD9744 模块 P1 为 2×10 排针：第 1 行为 CLK/PD，第 2～8 行为 D14～D1，
第 9 行为 GND/GND，第 10 行为 +5V/+5V。该排针与领航者 ZYNQ 的 40 针 RGB LCD
接口并不直接兼容，必须使用转接板或短排线，不能直接插接。建议所有数据线尽量等长，
时钟线旁安排地回流，并使用低纹波 5 V 电源；DAC、FPGA 和单片机必须共地。

## 六、模拟电压输出级

现有 AD9744 成品模块已经包含电流转电压、七阶重建滤波器和 OPA695 输出级。
内部基准下，模块空载输出约为 ±3 V，输出端串有 50 Ω 电阻，因此不要重复搭建这些
电路。比赛所需的高压输出级应接在模块 SMA 输出之后：

```text
AD9744 模块 SMA
    → 可选 49.9 Ω 输入端接
    → THS3491 同相放大器
    → 10 Ω 输出隔离电阻
    → 输出接口 / 1 kΩ 负载
```

建议第一版参数：

- THS3491 使用 ±15 V 电源，每个电源脚附近放置 100 nF 和 4.7 µF 去耦；
- 同相增益设置为约 4 倍；DDA 封装可从 `Rf≈798 Ω、Rg≈267 Ω` 起步；
- FPGA 幅度先设置为满量程的约 0.833，使最终峰值接近 ±10 V；
- 在单片机中保存增益和零偏校准系数，补偿模块及滤波器误差；
- 使用完整地平面，反馈回路必须短，DAC 到放大器使用 SMA/同轴连接；
- 5 MHz、10 V 峰值正弦波所需压摆率约为 314 V/µs；
- 基本要求的 1 kΩ 负载峰值电流为 10 mA。

发挥部分的 ±10 V/100 Ω 对应 100 mA 峰值，必须额外验证散热、限流、容性负载
稳定性和输出保护，不能直接按 1 kΩ 负载处理。

第一版可直接在 FPGA 中叠加直流偏置，因为成品模块信号链为直流耦合。波形峰值与
直流偏置之和不得超过校准后的 ±10 V 输出窗口。第二路电流源可复制一套 DAC 和调理
链路，再连接电压转电流模块；FPGA 内部可复用同一个采样时钟，增加第二套独立 DDS
寄存器和相位累加器。

## 七、推荐调试顺序

1. 执行 `create_clock_ip.tcl`，综合后确认 `clk_wiz_0/locked` 为高电平；
2. 暂不连接模拟输出级，测量 DAC CLK 是否为 100 MHz；
3. 将幅度设置为 0，确认 DAC 模块输出接近 0 V；
4. 先测试低幅度 1 kHz，再测试 1 MHz，最后扫频到 5 MHz；
5. 标定 AD9744 模块的增益和零偏；
6. 使用限流直流电源接入 ±15 V 高压驱动级和 1 kΩ 负载；
7. 测量幅度、偏置、失真、过冲、相位误差和器件温升。

当前 Codex 环境没有安装 Vivado，因此尚未重新生成 IP、执行综合实现或生成 bitstream。
必须在开发电脑上完成 Synthesis、Implementation 和 Generate Bitstream 后再下载测试。

## 八、行为仿真说明

工程的仿真顶层为 `tb_ad9744_dds_top`。行为仿真期间，`ad9744_clocking` 使用
100 MHz 等效时钟模型，因此即使尚未生成 Clocking Wizard 输出产品，也可以检查 DDS
和 DAC 接口逻辑。综合时会自动切换到真正的 `clk_wiz_0` IP 实例，不会把仿真时钟模型
综合进 FPGA。

测试平台会自动检查：

- `dac_sleep` 是否保持低电平；
- DAC 时钟周期是否约为 10 ns；
- 5 µs 内是否出现足够的 DAC 时钟边沿；
- 默认 1 MHz DDS 输出数据是否发生变化。

硬件综合使用工程内的 `clk_wiz_0.xci`。如果 Vivado 提示 IP 版本较旧，请先在
IP Status 窗口执行 Upgrade Selected，然后执行 Generate Output Products。
