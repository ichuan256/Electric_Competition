# LCR 元件测量系统项目概览

## 1. 功能需求

### 1.1 核心功能（必须实现）

1. 系统只提供 AUTO 测量模式。接入未知电阻、电容或电感后，自动判断元件类型并显示主要参数值，不提供手动指定 R、L、C 的模式。
2. 输出完整复阻抗结果：
   - 阻抗幅值 `|Z|`；
   - 相位 `phase`；
   - 串联等效电阻 `Rs`；
   - 电抗 `X`；
   - 电感 `L` 或电容 `C`。
3. 使用现有 AD9910 产生正弦激励，正常测量时直流偏置为 0。
4. Blue 同步采集电压和电流信息，不能再用单路电压幅值直接猜测元件值。
5. 每次测量先在 1 kHz～1 MHz 范围内进行多点粗扫，根据阻抗随频率变化的趋势判断 R、L、C 类型。
6. 对 L、C 自动找到使 `|Zx|` 最接近串联参考电阻 `Rref` 的频率；对 R 固定使用 500 kHz 精测。
7. 在选定频率下重复测量多次，对复数测量结果进行平均后输出最终值。
8. 支持自动选择参考电阻量程和安全激励幅值，整个过程不向用户暴露手动量程设置。
9. 支持 OPEN、SHORT、LOAD 校准并保存校准编号。
10. 在标定量程内达到题目要求的 1% 测量精度。
11. 测量失败、过量程、低信噪比或未校准时必须明确显示状态，不能把无效结果显示为 0。

### 1.2 次要功能（后续实现）

1. 显示 ESR、Q 值、损耗角正切 D、并联等效参数。
2. 保存粗扫曲线、最终结果和校准时间。
3. 对非理想元件显示阻抗随频率变化的简化曲线。

### 1.3 用户操作

1. 用户按 `B` 进入 LCR 页面。
2. 接入被测元件后按 `D` 开始自动测量。
3. Blue 自动完成粗扫、类型判断、精测频率选择和多次平均，用户不需要再选择类型或频率。
4. 屏幕依次显示粗扫进度、判定类型、精测频率、平均进度和最终结果。
5. 主结果以大字显示，例如 `C = 9.98 nF`，下方显示精测频率、ESR、相位、Rref档位和有效状态。

## 2. 非功能需求

### 2.1 精度

- 题目目标：L、C、R 测量误差不超过 1%。
- 建议首轮保证范围：
  - R：10 Ω～100 kΩ；
  - C：1 nF～100 µF；
  - L：100 µH～1 H。
- 扩展范围在完成夹具和多频点标定后再确定，不在未经验证时承诺 1%。

建议误差预算：

| 来源 | 目标上限 |
| --- | ---: |
| 参考电阻与温漂 | 0.15% |
| 两路模拟前端复增益残差 | 0.30% |
| ADC 噪声和重复性 | 0.20% |
| 夹具寄生与算法残差 | 0.30% |
| 总体设计目标 | ≤0.70%，给1%验收留余量 |

### 2.2 速度

- 单次4096点采样在2.5 MSPS下约1.6384 ms。
- 固定量程单点测量目标小于200 ms。
- 10点粗扫加8次精测平均的整次AUTO测量目标小于2 s。

### 2.3 可用性

- 测量过程中普通信号源配置不得覆盖当前激励。
- 任意异常必须恢复到安全激励幅值并释放测量状态机。
- 屏幕显示实际应用频率、实际量程和校准状态。

### 2.4 可维护性与扩展性

- 采集、复数拟合、阻抗计算、元件判定、标定和界面分别建模块。
- 所有物理量在协议中使用整数和明确单位，禁止跨节点传输浮点数。
- 保留原始两路复数相量用于调试和标定。

## 3. 约束与假设

1. Blue 是 LCR 测量主控，负责扫频状态机、自动判型、频率选择、量程开关、ADC同步采样、阻抗计算、平均和屏幕显示。
2. Black 只负责键盘事件转发，并按 Blue 的请求设置 AD9910 频率、幅值和启停状态。
3. 两路4096点ADC原始数据始终留在Blue本地处理，不通过Black↔Blue串口传输；最终结果由Blue直接送到LCD。
4. FPGA不参与基础LCR计算；后续可用于同步触发或高速采样扩展。
5. Blue 使用 `P1-9/PC0/ADC1_IN10` 采集 `Vin`，使用 `P1-8/PC1/ADC2_IN11` 采集 `Vr`；两脚均由开发板排针引出且未被 LCD 数据总线占用。
6. PC0 已不再连接或运行对数检波器，旧 `LogDetector` 与单 ADC/F9 测量模块不再加入运行工程，防止争用 ADC1、DMA 和串口解析器。
7. 两路输入都必须经过相同带宽、相同增益的偏置与保护前端，保持在 0～3.3 V；软件会分别拟合并去除直流分量。

## 4. 方案设计

### 4.1 系统结构

```text
Black键盘与AD9910执行器
        ▲
        │ Blue请求设置频率/幅值，Black回报实际FTW
        ▼
Blue LCR主状态机 ── 双ADC处理 ── 结果直接送LCD
        │
        ├── 1 kHz～1 MHz粗扫与自动判型
        ├── Rref自动量程和精测频率选择
        └── 多次采样、复数平均与R/L/C计算

物理测量链路：

GND ── Zx ── Vr节点 ── Rref ── Vin（AD9910激励）
          │                    │
       ADC通道B             ADC通道A
```

推荐测量拓扑：

```text
GND ── Zx ── Vr节点 ── Rref ── Vin
```

- ADC通道A测量 `Vin`。
- ADC通道B测量 `Vr节点` 对GND的电压；该电压就是被测元件 `Zx` 两端的电压。
- 参考电阻两端的复数电压为 `Vin - Vr`。
- 两路必须由同一个定时器边沿同步采样。

### 4.2 模拟前端

1. 两路采用尽量一致的衰减、偏置、运放缓冲和抗混叠网络。
2. 双极性交流信号偏置到ADC中点，建议约1.65 V。
3. ADC输入必须限制在0～3.3 V，并设置限流和钳位保护。
4. 测量激励从200 mVpp起步，可在50 mVpp～1 Vpp内自动调整；不得默认把7 Vpp大信号直接送入测量夹具。
5. 根据被测阻抗限制测试电流，建议首版不超过10 mA。
6. 参考电阻建议使用0.1%、低温漂器件，初始档位：

```text
10 Ω、100 Ω、1 kΩ、10 kΩ、100 kΩ
```

7. 量程切换优先使用低寄生继电器。若使用模拟开关，其导通电阻和寄生参数必须包含在每档复数标定中。

### 4.3 ADC采集

1. 使用STM32H750的ADC1和ADC2双规则同步模式。
2. TIM6 TRGO同时触发两路ADC。
3. DMA保存4096组同步样本，建议使用32位打包方式保存ADC1/ADC2数据。
4. 粗扫频率为1 kHz～1 MHz。采样率根据激励频率自动选择，并以2.5 MSPS为上限；每个频点必须覆盖足够周期，不能在1 kHz时仍固定使用过短的1.6384 ms窗口。
5. 粗扫每个频点采1帧；最终精测默认采8帧，并对复数结果做异常值剔除和平均。
6. 500 kHz～1 MHz频段必须单独验证ADC带宽、采样保持时间和前端相位误差。

### 4.4 复数相量计算

对每一路样本按实际激励频率和实际采样率拟合：

```text
y[n] = a*sin(2*pi*f*n/Fs) + b*cos(2*pi*f*n/Fs) + c
```

采用余弦参考定义复数相量：

```text
Y = b - j*a
```

分别得到：

```text
Vin = Bvin - j*Avin
Vr  = Bvr  - j*Avr
```

经过两路复增益标定后计算：

```text
Vref = Vin - Vr
I    = Vref / Rref
Vx   = Vr
Zx   = Vx / I
     = Rref * Vr / (Vin - Vr)
```

随后得到：

```text
R = real(Zx)
X = imag(Zx)
|Z| = sqrt(R*R + X*X)
phase = atan2(X, R)
```

元件参数采用串联等效模型：

```text
X > 0：L = X / (2*pi*f)
X < 0：C = -1 / (2*pi*f*X)
```

同时保留：

```text
电感Q = X / Rs
电容D = Rs / |X|
电容Q = 1 / D
```

### 4.5 元件类型判断

1. 粗扫使用1 kHz～1 MHz多个对数间隔频点，先获得 `|Z(f)|` 和相位曲线。
2. 对相邻有效点计算对数斜率，并取中位数抑制异常点：

```text
k = median(Δln|Z| / Δln f)
```

3. 结合斜率和相位自动判型：
   - `k > +0.35` 且相位主要为正：电感；
   - `k < -0.35` 且相位主要为负：电容；
   - `|k| < 0.20` 且相位接近0：电阻。
4. 电感的阻抗应随频率增大而增大；电容的阻抗应随频率增大而减小；电阻的阻抗应基本不随频率变化。
5. 处于阈值过渡区或多个频点结论矛盾时，增加复测点；仍不一致则显示“无法可靠分类”，不能强制输出R/L/C。

### 4.6 自动量程和频率

AUTO粗扫固定流程：

```text
1 kHz → 2 kHz → 5 kHz → 10 kHz → 20 kHz
→ 50 kHz → 100 kHz → 200 kHz → 500 kHz → 1 MHz
```

1. 粗扫阶段每点采1帧，用于获得趋势，不直接作为最终数值。
2. Rref档位由Blue内部自动选择，使 `0.1 ≤ |Zx|/Rref ≤ 10`；目标是 `|Zx|/Rref` 接近1。
3. 判定为电感或电容后，从有效粗扫点中选择：

```text
f_best = argmin |ln(|Zx(f)| / Rref)|
```

即选择 `|Zx|` 最接近串联参考电阻Rref的频率。必要时可根据相邻粗扫点插值，再落到已标定的频率表。
4. 判定为电阻后，不使用斜率最优点，固定在500 kHz下精测，并自动选择最接近电阻值的Rref档位。
5. 在最终频率下默认重复测量8次。先剔除明显异常帧，再平均两路复数相量或复阻抗，最后计算R、L或C；不得直接平均格式化后的显示数值。
6. 若任一ADC通道接近饱和，降低激励；若幅值过低，增大激励或切换Rref后重新测量该点。

### 4.7 校准

每个 `频率 × Rref档位` 保存一组复数校准参数：

- 参考电阻实际值；
- 两路通道复增益比；
- SHORT残余串联阻抗 `Zshort`；
- OPEN残余并联导纳 `Yopen`；
- 校准温度、时间和校准编号。

建议校准顺序：

1. SHORT：夹具短路，测量引线和开关的串联残余。
2. OPEN：夹具开路，测量输入和夹具的并联寄生。
3. LOAD：接入0.1%标准电阻，校准两路复增益比和量程比例。

初版修正模型：

```text
Z1 = Zraw - Zshort
Y1 = 1 / Z1 - Yopen
Zcorrected = 1 / Y1
```

LOAD用于求取每档复数比例系数。最终以标准R/C/L实测数据决定是否升级为完整三项误差模型。

### 4.8 通信协议

正式实现使用统一V2协议，不再以当前独立 `F9 26` 测电压协议作为最终接口。

LCR测量状态机和ADC数据处理均位于Blue。主要通信只用于键盘事件和AD9910激励控制：

| 命令 | 方向 | 用途 |
| --- | --- | --- |
| `KEY_EVENT 0x30` | Black→Blue | 转发B、D等键盘操作 |
| `LCR_EXCITATION_SET 0x48` | Blue→Black | 请求设置AD9910频率、幅值和启停 |
| `LCR_EXCITATION_READY 0x49` | Black→Blue | 回报实际FTW、实际频率和设置结果 |

Blue收到激励就绪响应并等待规定的稳定周期后，直接启动本地双ADC采集。原始数据和最终结果不需要经过Black，直接进入Blue屏幕状态。为了调试，可以在Blue内部保留：

```text
range_id、reference_resistor_mohm、calibration_id、
vin_complex、current_complex、result_flags
```

`MEASURE_RESULT 0x42` 可作为以后日志或上位机扩展，但不是本机LCD显示链路的必要环节。迁移完成后，旧 `F9 26` 单路测电压协议退出LCR主流程。

### 4.9 界面

LCR页面只保留AUTO操作，不显示类型、频率或量程的手动选择框：

1. 操作区：开始测量、返回、校准状态。
2. 进度区：当前粗扫频点、已完成点数、初步类型、精测频率和平均次数。
3. 主结果：自动判定的元件类型和主要数值，例如 `L = 10.02 mH`。
4. 详细结果：最终频率、Rref、|Z|、相位、ESR、Q/D、校准编号和状态。

状态文本统一为：

```text
COARSE_SWEEP → CLASSIFY → SELECT_BEST_FREQ
→ FINE_AVERAGE → DONE / ERROR
```

## 5. 当前系统与目标系统差距

| 项目 | 当前实现 | 目标实现 |
| --- | --- | --- |
| 激励 | 固定1 MHz满幅正弦 | Blue请求Black执行1 kHz～1 MHz自动扫频 |
| ADC | PC0/ADC1采Vin，PC1/ADC2采Vr，TIM6同步触发 | 完成模拟前端和双通道复增益标定 |
| 算法 | 4096组双ADC数据本地正弦拟合和复阻抗计算 | 增加校准补偿和异常帧剔除 |
| 输出 | RMS、Peak、Vpp | Blue本地直接显示R、L、C、ESR、Q、相位 |
| 量程 | 无 | 多档Rref自动量程 |
| 校准 | 仅假定3.3 V参考 | OPEN/SHORT/LOAD复数校准 |
| 协议 | 独立ADC测量V1帧 | 统一V2 LCR协议 |
| 精度 | 未形成LCR指标 | 标定范围内≤1% |

## 6. 实施阶段

### 阶段一：双通道硬件和原始相量

- 按 `P1-9=Vin`、`P1-8=Vr` 完成测量夹具和模拟前端接线。
- 用同一正弦信号验证两路同步 ADC 的幅值差和相位差。
- 屏幕显示两路幅值和相位差。
- 使用同一正弦信号分接两路，验证幅值比接近1、相位差接近0。

### 阶段二：单档阻抗测量

- 固定1 kΩ参考电阻和10 kHz激励。
- 实现复数Z、R、X、L、C计算。
- 使用100 Ω、1 kΩ、10 kΩ标准电阻验证公式和符号。

### 阶段三：自动分类和量程

- 增加Rref档位控制。
- 实现1 kHz～1 MHz粗扫、斜率自动判型和最佳频率选择。
- 实现电阻500 kHz精测及L/C最佳频率8次平均。
- 界面只保留AUTO测量，不增加手动R/L/C模式。

### 阶段四：校准与1%验收

- 实现OPEN/SHORT/LOAD校准存储。
- 使用标准R、C、L覆盖目标范围。
- 建立误差表，收缩或扩展最终承诺量程。

### 阶段五：统一协议和产品化界面

- 增加Blue→Black的AD9910激励设置/就绪协议，Blue本地保留完整LCR状态机。
- 完成状态、错误、扫频和校准页面。
- 移除或隔离旧的单路ADC测试入口。

## 7. 验收标准

1. 同一信号接入两路ADC时，校准后幅值差≤0.2%，相位差≤0.2°。
2. 标准电阻测量误差≤1%，且电抗接近误差预算范围内的0。
3. 标准电容测量误差≤1%，X为负，两个频点计算出的C一致。
4. 标准电感测量误差≤1%，X为正，两个频点计算出的L一致。
5. 连续测量10次的重复性优于0.3%。
6. OPEN、SHORT、过量程和低信噪比均返回明确无效状态。
7. 更换量程或频率后不会沿用上一档校准参数。
8. 测量完成后信号源和ADC均恢复到可控状态。
9. 标准L、C粗扫曲线的斜率方向正确，并能自动选择 `|Zx|` 最接近Rref的频率。
10. 标准电阻必须自动判定为R，并在500 kHz下完成最终8次平均。
11. ADC原始样本不经过板间UART，Blue处理完成后直接刷新LCD结果。

## 8. 待确认事项

1. 实际Rref档位、电阻精度、继电器/模拟开关型号和控制GPIO。
2. 首轮必须保证1%的R、C、L具体范围。
3. 实际测量激励经过哪一级放大器，以及夹具处可用的最大安全幅值。
4. 两路ADC前端的衰减、偏置、电压保护及500 kHz～1 MHz带宽。
5. 标准电阻、电容和电感的可用精度等级与数量。

## 9. 当前实施状态（2026-07-16）

已完成：

1. Blue 增加 AUTO LCR 主状态机和纯数学模块，包含十点粗扫、对数斜率自动判型、最佳频点选择、八次复阻抗平均以及 R/L/C 参数换算。
2. Blue→Black 增加 `LCR_EXCITATION_SET 0x48`，Black→Blue 增加 `LCR_EXCITATION_READY 0x49`；Black 在主循环中设置 AD9910 并回报实际 FTW。
3. D 键只触发 Blue 的 LCR 状态机，旧的 Black 主动发起 `F9 26` 单路电压测量已退出 D 键主流程。
4. LCR 页面直接读取 Blue 本地状态和结果，UART 不传 ADC 原始数组。
5. Blue 已实现 ADC1+ADC2 双规则同步采样：TIM6 TRGO 触发、DMA1 Stream2 保存4096组32位打包样本，并在Blue本地同时拟合 `Vin`、`Vr` 复相量。
6. PC0 已退出对数检波用途；Blue 旧 LogDetector、单ADC FFT/F9解析器和 Black 旧 F9客户端均已从运行工程移除。
7. LCR 数学模块已通过主机端测试；Blue、Black Keil 工程均通过 0 错误、0 警告重编译。

尚未完成：

1. 本机未安装 CubeMX，因此 ADC2/PC1、双模式、TIM6 和 DMA1 Stream2 目前由 `LcrDualAdc_User` 在运行时等效初始化；后续安装 CubeMX 后应把 PC1/ADC2 配置回填到 `.ioc`，再确认生成代码不与 User 驱动重复初始化。
2. Rref 自动切换 GPIO、每档实际阻值和 OPEN/SHORT/LOAD 校准仍依赖硬件确认。
3. 当前按理想相同通道计算，尚未加入两路模拟前端复增益校准；正式追求1%精度前必须用同源信号校正幅值比和相位差。
